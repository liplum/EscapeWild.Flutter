# 实体组件系统 (ECS)

本项目采用了一种轻量级的组件化架构，用于定义和扩展游戏实体的行为。

## 核心概念

### 1. 组件 (Component)

组件是纯粹的数据载体或特定行为的定义。

* **独立性**：每个组件通常只负责一个方面的功能（如：耐久度、新鲜度、可烹饪性）。
* **可序列化**：所有组件都支持序列化，以便于游戏的存档和读档。

### 2. 组件混入 (CompMixin)

这是一种将组件附加到对象上的机制。在本项目中，`Item`（物品原型）是主要的组件宿主。

* **多组件支持**：一个实体可以拥有多个同类型或不同类型的组件。
* **动态查询**：系统可以在运行时查询实体是否具有特定组件，并获取其数据。

## 组件的应用场景

### 物品行为定义

通过为物品添加不同的组件，可以定义其在游戏中的特性：

* **耐久组件**：定义物品的使用寿命。
* **新鲜度组件**：定义食物随时间腐烂的逻辑。
* **工具组件**：定义物品作为某种工具（如斧头、小刀）的效能。
* **属性修改组件**：定义使用该物品后对玩家属性（如饥饿值、生命值）的影响。

### 逻辑解耦

ECS 架构使得游戏逻辑不再依赖于复杂的继承树。例如，一个“带水的瓶子”既是一个“容器”，也是一个“消耗品”，这通过同时挂载 `ContainerComp` 和 `UsableComp` 即可实现，而无需创建一个复杂的子类。

## 扩展性

开发者可以通过实现新的组件类来轻松添加新的游戏机制，而无需修改现有的核心代码。
