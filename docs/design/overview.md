# 游戏设计架构概述

本项目是一个荒野求生类游戏的后端逻辑框架。它采用高度模块化的设计，旨在模拟复杂的生存环境、物品交互和玩家状态管理。

## 核心设计理念

1. **数据驱动**：游戏中的物品、配方和场景主要通过配置和组件定义，而非硬编码逻辑。
2. **组件化 (ECS)**：利用实体组件系统（Entity Component System）的思想，将功能拆分为可复用的组件，附加到物品或实体上。
3. **真实模拟**：包含时间流逝、物品耐久、新鲜度、环境湿度等模拟要素，增强生存体验。

## 主要系统模块

* **实体组件系统 (ECS)**：提供基础的组件管理机制，支持功能的动态扩展。
* **物品与背包系统**：管理物品的属性（质量、堆叠、容器属性）以及玩家的负重。
* **玩家状态系统**：追踪玩家的各项生理指标（如生命值、饥饿度等）和地理位置。
* **探索与路线系统**：定义游戏世界的结构，包括不同的地理区域、地点以及在这些地点可以执行的动作。
* **合成与烹饪系统**：处理资源的转化，包括工具制作、食物加工和营火交互。
* **时间与环境系统**：驱动游戏世界的运行，处理随时间推移产生的状态变化（如食物腐烂、属性消耗）。

## 游戏流程简述

玩家在特定的**关卡 (Level)** 中开始游戏，通过在不同的**地点 (Place)** 执行**动作 (Action)**（如采集、移动、烹饪）来消耗**时间 (Time)**。动作的结果会影响玩家的**属性 (Attributes)** 和**背包 (Backpack)** 中的物品。玩家的目标是在资源耗尽前完成预设的生存目标或到达终点。
