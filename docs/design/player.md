# 玩家属性与状态

玩家系统负责追踪幸存者的生理状态、位置信息以及游戏进度。

## 核心属性 (Attributes)

玩家的状态由一系列数值化的属性组成，常见的包括：

* **生命值 (Health)**：玩家的生存基础，归零则游戏结束。
* **饥饿度 (Hunger)**：随时间消耗，通过进食补充。
* **口渴度 (Thirst)**：随时间消耗，通过饮水补充。
* **精力 (Energy)**：执行动作会消耗，通过休息补充。

### 属性修正机制

* **难度修正**：游戏难度会影响属性消耗的速度或恢复的效率。
* **环境影响**：如湿度、温度等环境因素可能加速某些属性的下降。

## 状态管理

### 1. 生死判定

系统实时监控生命值。当生命值降至 0 时，触发游戏失败逻辑。

### 2. 位置追踪

记录玩家当前所处的**地点 (Place)**。位置决定了玩家当前可以执行哪些动作（如在河边可以钓鱼，在森林可以采集）。

### 3. 进度追踪

* **行动次数**：记录玩家执行的总动作数。
* **生存时间**：记录游戏世界内流逝的总时间。
* **旅程进度**：衡量玩家距离最终目标的完成程度。

## 动作执行 (Action)

玩家通过执行动作与世界交互。

* **动作消耗**：大多数动作会消耗时间，并可能导致属性变化。
* **前提条件**：某些动作需要特定的工具（如砍树需要斧头）或处于特定的地点。
* **自动工具选择**：当执行需要工具的动作时，系统会根据玩家的偏好或背包中工具的效能自动选择最佳工具。
